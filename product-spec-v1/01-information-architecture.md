# 01 · 信息架构与页面清单

## 1. 产品结构

V1 采用“公开入口 + 单一高频工作台 + 低频资产/管理页”的结构。任务状态作为全局抽屉，不额外制造页面跳转。

```text
Cornfield
├─ 落地页 /                             P0，公开预渲染页
├─ 登录 /app/login                      P0
├─ 首次改密 /app/change-password         P0
├─ 创作 /app/create                     P0，登录后默认页
│  ├─ 灵感墙（最近生成）
│  ├─ 生成器（固定悬浮）
│  ├─ 图片预览层
│  └─ 任务抽屉（全局覆盖层）
├─ 资产 /app/assets                     P0
│  ├─ 全部图片
│  ├─ 搜索与筛选
│  └─ 图片预览层
└─ 管理 /app/admin                      P0，仅管理员
   ├─ 用户 /app/admin/users
   ├─ 模型 /app/admin/models
   └─ 供应商 /app/admin/providers
```

V1 不设置公开 Explore、社区、评论、点赞、积分、套餐、账单、邀请、视频、Draw、局部重绘、放大增强等入口。

## 2. 全局导航

桌面端使用 56px 顶栏，内容为：

- 左侧：产品标识，点击回到 `/app/create`。
- 中部：`创作`、`资产` 两个一级入口。
- 右侧：任务状态入口、设置入口、内部身份缩略信息。

生成中的任务不会因为切换页面而中断。任务入口显示“运行中数量”，而非红点式营销提醒。

## 3. 页面清单

| 页面/覆盖层 | 路由 | 优先级 | 用户目标 | 核心内容 | 关键状态 |
|---|---|---:|---|---|---|
| 落地页 | `/` | P0 | 了解并进入工具 | 品牌叙事、产品视觉、登录入口 | 静态预渲染、响应式 |
| 登录/首次改密 | `/app/login`、`/app/change-password` | P0 | 建立安全会话 | 用户名密码、一次性密码更新 | 错误、限速、密码过期 |
| 图片生成页 | `/app/create` | P0 | 生成、查看、复用图片 | 灵感墙、生成器、缩放器 | 首次空态、加载、生成中、部分失败、离线 |
| 资产库 | `/app/assets` | P0 | 找回历史作品 | 搜索、模型/比例/日期筛选、瀑布流 | 空态、无结果、分页失败 |
| 图片预览 | 覆盖层 | P0 | 看大图并继续创作 | 大图、提示词、参数、参考/复制/下载 | 原图加载、复制失败、下载失败 |
| 任务抽屉 | 全局覆盖层 | P0 | 了解当前与最近任务 | 批次、子任务、进度、错误、重试 | 排队、运行、成功、部分成功、失败 |
| 用户管理 | `/app/admin/users` | P0 | 管理内部账号 | 创建、停用、重置密码、审计 | 临时密码、最后管理员保护 |
| 模型设置 | `/app/admin/models` | P0 | 查看静态模型配置状态 | 启用状态、排序、能力 revision 与 drift（只读） | 能力冲突、停用、待部署 |
| 供应商设置 | `/app/admin/providers` | P0 | 检查接入状态 | Legnext/OpenRouter 状态、熔断、最近错误 | 可用、限流、鉴权失败、不可用 |

供应商设置页只展示 enabled/state、熔断截止、最近探测/错误和 active job 数；不返回密钥是否存在、末四位、完整 API Key 或任何可用于推断凭证的片段。

## 4. 图片生成页信息层级

从高到低分为四层：

1. **全局层**：导航、任务状态、管理员入口。
2. **内容层**：最近作品瀑布墙，承担结果反馈和灵感回看。
3. **创作层**：固定在视口底部的生成器，是页面唯一主操作。
4. **聚焦层**：图片预览、模型选择器、比例选择器等覆盖层。

生成器始终可达，但不会挡住瀑布流最后一行：内容区底部预留生成器高度 + 40px 安全区。

## 5. 主用户旅程

### 5.1 文生图

```text
进入创作页 → 查看最近作品 → 输入提示词 → 选择模型/比例/分辨率/抽卡次数
→ 提交 → 生成器立即清空或保留（按个人偏好） → 占位卡进入灵感墙
→ 图片逐张可见 → 悬浮参考/复制/下载
```

### 5.2 图生图

```text
悬浮历史图片 → 点击“参考” → 图片进入生成器参考区
→ 修改提示词与参数 → 提交 → 新结果保留来源关系
```

也允许从本地上传参考图。上传完成前可以编辑其他参数，但不能提交。

### 5.3 找回资产

```text
进入资产页 → 搜索提示词或按模型/日期/比例筛选
→ 打开预览 → 下载，或带回创作页作为参考图
```

## 6. 图片卡片操作规则

桌面端悬浮时只显示三个一级动作：

1. **参考**：把该图片加入当前生成器的参考图列表；成功后给轻量反馈。
2. **复制**：优先复制图片二进制到系统剪贴板；浏览器不支持时降级为复制内部可访问链接。
3. **下载**：下载原图；文件名由日期、模型短名、资产 ID 组成。

点击图片非按钮区域打开预览。移动端无 hover，单击图片打开预览，三个动作固定在预览底部。

## 7. 导航与权限边界

- 工具部署在公司 VPN/Zero Trust 后仍必须经过应用自身的用户名+密码登录；不信任客户端或代理传入的身份头。
- V1 无公开注册、邮件邀请/找回或复杂 RBAC。首个管理员由 CLI 创建；管理员可创建、停用、重置内部账号，临时密码 24 小时有效且首次登录强制修改。
- `创作`、`资产` 对 active 内部用户开放；`/app/admin/*` 仅管理员可见。禁止停用或降级最后一个有效管理员。
- 会话使用 HttpOnly/Secure/SameSite Cookie，修改请求要求 CSRF token；停用账号、重置密码或增加 `session_version` 后，数据库 session 与活跃 SSE 必须立即失效。

## 8. 路由参数与可分享状态

- `/app/create?model={model_id}`：预选模型。
- `/app/create?reference={asset_id}`：将单个当前用户资产带入参考区。
- `/app/assets?model=&ratio=&from=&to=&q=`：保留筛选状态。
- 图片预览不暴露公开 URL；资产内容始终通过 `/api/v1/assets/{asset_id}/content` 鉴权，并由 Nginx 内部跳转直出。

提示词草稿、最后使用的模型、比例、分辨率、抽卡次数和墙面缩放级别保存在本地；任务与资产以服务端为准。
